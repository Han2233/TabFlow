# TabFlow - 智能标签页管理工具 需求文档

> **版本**: V1.0  
> **日期**: 2026-06-29  
> **产品形态**: Chrome 扩展 + 后端服务  
> **技术栈**: 前端 React + TypeScript / 后端 Python FastAPI  

---

## 一、背景与痛点

日常开发中，开发者经常同时打开大量浏览器标签页（Aone 需求页、代码仓库、接口文档、设计稿、各类后台系统等），导致：

1. **标签页拥挤**：标签栏宽度被压缩到无法辨认标题，难以快速找到目标页面
2. **缺乏组织**：不同任务、不同类型的页面混在一起，没有清晰的逻辑分组
3. **上下文切换成本高**：切换不同需求/任务时，需要在大量标签页中逐一查找关联页面
4. **内存消耗大**：大量标签页长期驻留导致浏览器占用过多内存，影响开发体验
5. **工作状态难以恢复**：浏览器崩溃或重启后，之前精心组织的标签页布局丢失

## 二、产品定位

**TabFlow** 是一款面向开发者的智能标签页管理 Chrome 扩展，通过 AI 驱动的智能分类、工作区快照、一键分窗等核心能力，帮助开发者高效组织和管理浏览器标签页，降低上下文切换成本，提升开发效率。

## 三、目标用户

- 日常工作中需要同时打开 20+ 标签页的技术开发者
- 需要在多个需求/任务之间频繁切换的开发工程师
- 希望优化浏览器内存占用、提升开发体验的用户

## 五、功能详细设计

### 5.1 智能分组管理

#### 5.1.1 按域名/站点自动聚合

**描述**：根据标签页 URL 的域名自动将标签页归入对应分组。

**规则示例**：
| 域名模式 | 分组名称 | 颜色标识 |
|---------|---------|---------|
| `*.aone.alibaba-inc.com` | Aone | 蓝色 |
| `github.com/*`, `gitlab.com/*` | 代码仓库 | 绿色 |
| `yuque.antfin.com/*`, `docs.google.com/*` | 文档 | 黄色 |
| `localhost:*`, `127.0.0.1:*` | 本地开发 | 紫色 |

**功能要点**：
- 系统内置常见开发站点的默认分组规则
- 用户可自定义域名匹配规则和分组名称
- 支持通配符和正则表达式匹配
- 新打开的标签页自动匹配规则并归组

#### 5.1.2 按需求/任务手动分组

**描述**：用户可以创建自定义分组（如"XX 需求开发"），将相关标签页拖拽归入。

**功能要点**：
- 创建/编辑/删除自定义分组，支持命名和颜色标识
- 拖拽标签页到分组中，支持批量选择和拖拽
- 分组支持添加备注（如关联的需求 ID、任务链接等）
- 分组内标签页支持排序（手动排序 / 按标题 / 按打开时间）

#### 5.1.3 按时间维度分组

**描述**：根据标签页打开时间自动生成时间维度的分组视图。

**功能要点**：
- 支持"今天"、"昨天"、"本周"、"更早"等时间分组
- 可用于快速清理过期标签页（如一键关闭"更早"分组的所有标签页）
- 作为辅助视图存在，与其他分组方式可切换使用

#### 5.1.4 AI 智能分类

**描述**：调用外部 AI API，基于标签页的标题、URL、页面摘要等信息进行语义分析，自动为标签页推荐分组。

**功能要点**：
- 用户点击"AI 整理"按钮，触发 AI 分类（非实时自动触发，避免不必要的 API 调用）
- AI 分析当前所有未分组标签页，输出分组建议（分组名称 + 标签页归属）
- 用户可预览 AI 分类结果，确认后一键应用，也可手动调整后再应用
- 支持用户对 AI 分类结果进行反馈（正确/错误），用于后续优化提示词
- AI 分类时会参考用户已有的自定义分组，优先归入已有分组

**AI 分类策略**：
- 输入：标签页标题、URL、域名、页面 meta description
- 上下文：用户已有分组、历史分类偏好
- 输出：建议分组名 + 置信度
- 后端通过 Prompt Engineering 调用大模型 API（如通义千问 / Claude）

### 5.2 工作区管理

#### 5.2.1 一键分窗口打开

**描述**：将当前的标签页分组按照每个分组一个窗口的方式，自动创建多个浏览器窗口并分配标签页。

**功能要点**：
- 点击"分窗口打开"后，为每个分组创建一个独立的浏览器窗口
- 每个窗口的标签页按分组内的顺序排列
- 支持选择性分窗（选中部分分组进行分窗，而非全部）
- 分窗后原窗口的处理策略：可选保留或关闭
- 窗口创建时自动应用 Chrome Tab Groups 的颜色和标签

#### 5.2.2 工作区快照保存

**描述**：将当前所有窗口和标签页的布局保存为一个"工作区快照"，方便后续一键恢复。

**功能要点**：
- 保存当前所有窗口的标签页 URL、分组信息、窗口位置和大小
- 支持为快照命名（如"XX 需求联调环境"、"线上问题排查"）
- 快照列表展示，支持搜索和排序
- 快照支持编辑（添加/删除 URL、修改分组）
- 数据存储在本地（chrome.storage.local），V1 不做跨设备同步

#### 5.2.3 工作区快照恢复

**描述**：从已保存的快照中恢复窗口和标签页布局。

**功能要点**：
- 恢复时可选择"在新窗口中恢复"或"替换当前窗口"
- 恢复时自动跳过已打开的标签页（避免重复）
- 支持部分恢复（只恢复快照中的部分分组）

### 5.3 标签页搜索

**描述**：提供全局搜索能力，通过关键词快速定位到目标标签页。

**功能要点**：
- 搜索范围：标签页标题、URL
- 实时搜索，输入即过滤，高亮匹配关键词
- 搜索结果点击后直接跳转到对应标签页（激活该标签页并切换到其所在窗口）
- 支持快捷键唤起搜索（默认 `Ctrl+Shift+F` / `Cmd+Shift+F`）
- 搜索结果展示分组归属信息

### 5.4 重复标签检测与合并

**描述**：自动检测并提示用户合并相同 URL 的重复标签页。

**功能要点**：
- 后台定期扫描（可配置间隔或手动触发）检测重复标签页
- 检测维度：完全相同的 URL / 忽略 URL 参数后相同 / 同一页面不同锚点
- 发现重复后在扩展图标上显示角标提示
- 用户可查看重复列表，选择保留哪个、关闭哪些
- 支持一键合并所有重复标签页（保留最近访问的那个）

### 5.5 标签页休眠/挂起

**描述**：自动或手动将长时间未访问的标签页挂起，释放内存资源。

**功能要点**：
- 自动休眠：超过可配置时间（默认 30 分钟）未访问的标签页自动挂起
- 手动休眠：支持手动挂起指定标签页或指定分组
- 休眠状态展示：休眠的标签页在侧边栏中有明显的视觉标识（如灰色图标）
- 唤醒方式：点击休眠标签页时自动重新加载
- 白名单：支持设置不自动休眠的域名（如正在调试的 localhost）
- 内存节省统计：展示通过休眠节省的内存量

### 5.6 关闭历史 / 标签页回收站

**描述**：自动记录所有被关闭的标签页，保留一段时间内的关闭历史，支持随时恢复。覆盖两类典型场景：误关恢复、任务完成后因 Bug 需要回溯工作上下文。

**功能要点**：
- 自动记录：所有被关闭的标签页（包括单个关闭、批量关闭、关闭整个窗口）自动进入关闭历史
- 保留分组信息：如果关闭的是整个分组或窗口，保留当时的分组结构，恢复时可按组一键恢复
- 保留时长：默认 30 天，用户可在设置中自定义（7 天 / 14 天 / 30 天 / 90 天 / 永久）
- 存储上限：设置最大记录条数（默认 5000 条），超出后自动清理最早的记录
- 自动清理：到期记录自动清理，也支持手动清空

**恢复能力**：
- 单个恢复：从历史列表中点击恢复单个标签页
- 按组恢复：如果一批标签页是同一次操作关闭的（如关闭整个分组），支持一键恢复整组
- 按时间段恢复：选择某个时间点，恢复该时间点之后关闭的所有标签页
- 恢复时自动跳过当前已打开的相同 URL（避免重复）

**历史列表展示**：
- 按时间倒序排列，显示关闭时间、标签页标题、URL、来源分组
- 支持搜索和筛选（按关键词、按分组、按时间范围）
- 批量关闭的记录聚合展示（如"XX 需求分组 - 5 个标签页"），可展开查看明细
- 在侧边栏中作为独立 Tab 或可切换的视图存在

**与其他功能的关系**：
- 与"工作区快照"互补：快照是主动保存的完整工作区状态，关闭历史是被动自动记录的安全网
- 休眠标签被手动关闭后同样进入关闭历史

### 5.7 侧边栏面板（Side Panel）

**描述**：利用 Chrome Side Panel API 提供一个常驻的标签页管理面板，作为主要的交互入口。

**面板布局**：
```
┌──────────────────────────┐
│ 🔍 搜索标签页...          │
├──────────────────────────┤
│ [分组] [时间] [历史] [全部]  │
├──────────────────────────┤
│ ▼ 🔵 Aone (5)            │
│   ├ 需求详情 - XXX        │
│   ├ 任务列表              │
│   └ ...                  │
│ ▼ 🟢 代码仓库 (3)         │
│   ├ PR #1234             │
│   └ ...                  │
│ ▶ 🟡 文档 (8) 💤          │
│ ▼ ⚪ 未分组 (12)          │
│   └ ...                  │
├──────────────────────────┤
│ [AI整理] [分窗口] [保存快照]│
│ 💾 已节省内存: 1.2 GB      │
└──────────────────────────┘
```

**功能要点**：
- 树形结构展示所有分组和标签页
- 分组支持折叠/展开
- 支持拖拽排序和跨分组移动
- 右键菜单：关闭、休眠、移动到分组、复制 URL 等
- 底部操作栏：AI 整理、分窗口打开、保存快照等核心操作
- 分组/时间/历史/全部四种视图切换
- 显示内存节省统计

### 5.8 Popup 快捷操作

**描述**：点击扩展图标弹出的小窗口，提供轻量级快捷操作。

**功能要点**：
- 快速搜索栏
- 当前窗口标签页数量和分组概览
- 常用操作快捷按钮：AI 整理、保存快照、检测重复、打开侧边栏
- 引导用户打开侧边栏进行更复杂的管理操作

### 5.9 快捷键

| 快捷键 | 功能 |
|-------|------|
| `Ctrl+Shift+F` | 唤起标签页搜索 |
| `Ctrl+Shift+S` | 保存当前工作区快照 |
| `Ctrl+Shift+G` | 打开/关闭侧边栏面板 |
| `Ctrl+Shift+A` | 触发 AI 智能整理 |

*快捷键支持用户自定义修改*

### 6.2 扩展端技术方案

| 组件 | 技术选型 | 说明 |
|-----|---------|------|
| UI 框架 | React 18 + TypeScript | 侧边栏和 Popup 共用组件库 |
| 构建工具 | Vite + CRXJS | 专为 Chrome 扩展优化的 Vite 插件 |
| 状态管理 | Zustand | 轻量级，适合扩展场景 |
| 样式方案 | Tailwind CSS | 快速开发，样式隔离好 |
| 本地存储 | chrome.storage.local | 存储分组规则、快照、用户配置 |
| 扩展 API | Chrome Extensions Manifest V3 | 使用 Service Worker 作为后台脚本 |

### 6.3 后端技术方案

| 组件 | 技术选型 | 说明 |
|-----|---------|------|
| Web 框架 | Python FastAPI | 异步高性能，自动生成 API 文档 |
| AI 集成 | LangChain / 直接调用 API | 调用通义千问或 Claude API |
| 数据库 | SQLite（V1）| V1 单机部署，后续可切换 PostgreSQL |
| 数据模型 | SQLAlchemy + Alembic | ORM + 数据库迁移 |
| 部署 | Docker | 容器化部署 |

### 6.4 核心 API 设计

```
POST   /api/v1/classify          # AI 分类标签页
POST   /api/v1/classify/feedback  # 提交分类反馈
GET    /api/v1/rules              # 获取用户分组规则
POST   /api/v1/rules              # 创建分组规则
PUT    /api/v1/rules/{id}         # 更新分组规则
DELETE /api/v1/rules/{id}         # 删除分组规则
```

## 七、非功能需求

### 7.1 性能要求
- 侧边栏面板打开时间 < 200ms
- 标签页搜索响应时间 < 50ms（本地）
- AI 分类请求响应时间 < 5s（含网络延迟）
- 内存占用增量 < 50MB（扩展本身）

### 7.2 兼容性
- 支持 Chrome 116+（Side Panel API 最低版本）
- 支持 Chromium 系浏览器（Edge、Brave 等）

### 7.3 数据安全
- 标签页数据仅在本地和用户自部署的后端之间传输
- AI 分类仅发送标题和 URL，不发送页面内容
- 后端服务支持自部署，数据不经过第三方

### 7.4 用户体验
- 首次安装后提供引导教程
- 所有操作支持撤销（Ctrl+Z）
- 分组颜色与 Chrome 原生 Tab Groups 颜色保持一致
